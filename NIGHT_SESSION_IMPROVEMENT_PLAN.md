# 夜盤交易檢討總結與改進方案

## 📊 檢討結果摘要

### 發現問題
1. **策略配置不一致**: 配置為 `options_squeeze`，實際執行 `ThetaGang iron_condor`
2. **過度交易**: 夜盤48筆交易，平均持倉1-3分鐘
3. **出場機制單一**: 僅依賴波動率擴張信號出場
4. **數據品質問題**: K線間隔異常，數據更新延遲

### 已實施改進
1. **策略配置統一**: 修正為 `v2_squeeze`
2. **交易頻率控制**: 增加冷卻時間(20根K線)和最小持倉時間(10根K線)
3. **出場機制強化**: 啟用時間衰減(theta)出場，設定停損停利(5%/30%)
4. **數據監控**: 創建數據品質檢查腳本

## 🔧 技術修正詳情

### 1. 配置修正
```yaml
# config/options.yaml (修正前)
strategy: "options_squeeze"

# config/options.yaml (修正後)  
strategy: "v2_squeeze"  # 與實際插件名稱一致
```

### 2. 參數優化
```yaml
# config/options_strategy.yaml (新增)
theta_gang:
  cooldown_bars: 20          # 冷卻時間增加
  min_holding_bars: 10       # 最小持倉時間
  exit_on_theta: true        # 時間衰減出場
  stop_loss_pct: 0.05        # 5%停損
  take_profit_pct: 0.3       # 30%停利
```

### 3. 監控工具
- **數據品質檢查腳本**: `scripts/data_quality_monitor.py`
- **自動檢測**: K線間隔、數據新鮮度、價格有效性
- **日誌記錄**: 保存至 `logs/data_quality.log`
