1. 均值回歸策略 (Mean Reversion)
核心邏輯：認為價格偏離平均值過多時，最終會回跌或回升。
指標：使用 布林通道 (Bollinger Bands)。
自動化條件：
買入：當零股價格觸碰或低於「下軌」時，買入少量股數。
賣出：當價格回升至「中軌」或「上軌」時平倉。
優點：適合在大型權值股（如 2330, 2454）橫盤震盪時賺取小額價差。

2. 動能突破策略 (Momentum Breakout)
核心邏輯：追蹤盤中強勢股，當零股價格突破當日高點時跟進。
指標：當前價 > 開盤價 * 1.02 (漲幅 2%) 且 成交量放大。
自動化條件：
買入：突破盤整區間上緣。
賣出：跌破移動止損點（例如最高價回落 1.5%）。
注意：零股流動性較差，突破時可能會有滑價（Slippage）風險。
3. 分批定額買入 (Smart DCA)
核心邏輯：將大額資金拆分，利用自動化腳本在特定時間或價位買入，平滑成本。
策略設計：
時間觸發：每天 11:00 準時買入固定金額（例如 3,000 元）的零股。
價位優化：如果當前價低於昨日收盤價，則加碼買入 1.5 倍股數。
適合對象：長期看好高價股但預算有限的小資族。

4. 整股/零股價差監控 (Arbitrage Lite)
核心邏輯：監控「整股」與「零股」之間的溢價/折價。
自動化條件：
當 零股折價（零股價 < 整股價 * 0.995）時，自動買入零股。
持有目標：長期持有或等待價差收斂後賣出。
優點：這是一種相對低風險的策略，因為兩者本質是同一家公司的股票。

A. 隔日沖 (Swing Trading)
策略：今天收盤前監控 Squeeze 指標，若觸發買進訊號則買入。
執行：持有至少到「隔天」或「數天後」達到目標價才賣出。
優點：符合零股交易規則，且能利用 API 自動監控多檔標的。
B. 自動化存股 / 定期定額
策略：監控技術指標（如 RSI 低於 30 或 KD 低位金叉）。
執行：當價格「相對便宜」時，自動買入固定股數。

🕒 1. 核心邏輯架構
監控時間：使用 datetime 檢查現在是否到達 13:20。
撤銷未成交單：將所有掛出去但還沒買到/賣掉的委託單取消（Cancel）。
強制市價/限價平倉：檢查目前庫存（昨日留倉），若未達獲利目標也強制賣出。
import datetime
import time
import shioaji as sj

def time_based_exit(api):
    now = datetime.datetime.now().time()
    exit_time = datetime.time(13, 20) # 設定止損時間

    if now >= exit_time:
        print(f"⏰ 當前時間 {now}，觸發時間止損流程...")

        # --- 第一步：撤銷所有「掛單中」的委託 ---
        api.update_status() # 更新訂單狀態
        for trade in api.trades:
            if trade.status.status in [sj.constant.OrderStatus.PreSubmitted, 
                                       sj.constant.OrderStatus.Submitted, 
                                       sj.constant.OrderStatus.PartiallyFilled]:
                print(f"🚫 撤銷未完成委託單: {trade.contract.code}")
                api.cancel_order(trade)

        # --- 第二步：強制平倉現有庫存 (如果是要做隔日沖的平倉) ---
        # 取得目前帳戶餘額/庫存
        positions = api.list_positions(api.stock_account)
        for pos in positions:
            if pos.quantity > 0:
                print(f"📉 強制平倉庫存: {pos.code}, 數量: {pos.quantity}")
                
                # 建立賣出委託 (建議用跌停價或目前買進價以求立即成交)
                contract = api.Contracts.Stocks[pos.code]
                order = api.Order(
                    price=0, # 若支援市價可用 0，零股建議用當前買盤一價
                    quantity=int(pos.quantity),
                    action=sj.constant.Action.Sell,
                    price_type=sj.constant.StockPriceType.MKT, # 市價單
                    order_type=sj.constant.OrderType.ROD,
                    order_lot=sj.constant.StockOrderLot.Odd # 零股平倉
                )
                api.place_order(contract, order)
        
        return True # 已執行止損
    return False

⚠️ 零股平倉的特殊限制
無法「現買現賣」：
如果您是 今天才買入 的零股，13:20 執行這段程式碼會報錯（因為不能當沖）。
此邏輯僅適用於 「昨天買，今天賣」 的隔日沖策略。
流動性風險：
13:20 接近收盤，零股撮合可能非常緩慢。
建議做法：不要使用 MKT (市價)，而是使用 LMT (限價) 並掛 「買進第一檔」 或 「跌停價」，以增加成交機會。
撤單的重要性：

🛠️ 如何在同一個 API 腳本中整合？
為了避免衝突，建議你在程式碼中引入 「策略分類 (Strategy Tagging)」。
1. 資金分帳 (Virtual Sub-accounts)
雖然你只有一個永豐金帳號，但你在 Python 程式內可以定義兩組清單：
List A (Speculation)：專注於 Squeeze 噴發股（如：熱門強勢股 2330, 2317）。
List B (Investment)：專注於長期看好且有套利空間的權值股（如：0050 成分股）。
2. 下單註記與日誌區分
在執行下單時，你的 Log 必須清楚標示來源：

⚠️ 最嚴重的衝突：13:20 強制止損
這是你目前程式中最可能出錯的地方。
問題：如果你 13:20 執行 cancel_all_orders 或 list_positions 強制賣出，你會連「長線套利」的股票也一起賣掉。
解決方案：
方案 A (推薦)：在程式中建立一個 Speculation_Inventory 的資料表（例如 CSV 或 SQLite），紀錄哪些股票是為了「短線爆發」買的。
13:20 邏輯改寫：只賣出紀錄在 Speculation_Inventory 裡的股票，不碰其他持股。




🛠️ 「零股偵察兵」策略邏輯架構
這個策略可以拆解為三個階段：試單 (Test)、確認 (Confirm)、擴大 (Scale)。
1. 第一階段：零股試單 (The Scout)
觸發條件：Squeeze 指標擠壓後剛開始噴發，或價格突破關鍵壓力位。
動作：買入小額零股（例如 10~50 股）。
目的：建立「心理倉位」，讓程式開始追蹤這檔股票的即時表現。
2. 第二階段：趨勢過濾與停損 (The Filter)
失敗判定：買入後價格跌破前一根 K 線低點，或虧損超過 1.5%。
動作：果斷停損。因為是零股，賠掉的絕對金額極小（可能只有幾十塊手續費）。
意義：用極低代價避開了「假突破」。
3. 第三階段：強勢加碼 (The Heavy Hit)
成功判定：獲利達到 1%~2%，且成交量持續放大，股價站穩。
動作：觸發二次加碼（買入整股 1000 股或更多零股）。
目標：此時你的平均成本雖然提高，但勝率大幅上升。
# 策略紀錄字典 (虛擬快取)
active_trades = {
    "2330": {"stage": "SCOUT", "entry_price": 1000, "qty": 10},
}

def monitor_and_scale(code, current_price):
    trade = active_trades.get(code)
    if not trade: return

    profit_pct = (current_price - trade['entry_price']) / trade['entry_price']

    # --- 1. 偵察兵失敗：停損 ---
    if profit_pct < -0.015: 
        logger.warning(f"❌ {code} 趨勢錯誤，零股偵察兵撤退 (停損)")
        # api.place_order(Sell...)
        del active_trades[code]

    # --- 2. 偵察兵成功：加碼 (擴大獲利) ---
    elif trade['stage'] == "SCOUT" and profit_pct > 0.01:
        logger.info(f"🔥 {code} 趨勢確認！執行大額加碼...")
        # api.place_order(Buy, quantity=1000, order_lot=Common)
        trade['stage'] = "MAIN_FORCE"
        trade['entry_price'] = current_price # 更新成本或紀錄加碼點
📈 為什麼這個策略適合「短線爆發隔日沖」？
解決「不敢下單」的問題：零股金額小，程式自動執行時你不會心痛。
汰弱留強：你的帳戶裡留下的永遠是「正在獲利」的強勢股，賠錢的爛股票在零股階段就被清掉了。
期望值正向：
賠錢時：賠的是 10 股的 1.5% (極小)。
賺錢時：賺的是 1010 股的 3%~5% (極大)。
這就是典型的 「大賺小賠」 數學模型。
A. 加碼後不要忘記 13:20 規則
加碼後的股票（整股）如果是在「今天」買的，可以當沖。
如果 13:20 獲利不如預期，整股部分可以執行 「現股當沖賣出」。
零股部分（早上買的）則必須留到 「隔天」 賣出。
B. 手續費陷阱
因為「試單」頻率可能很高，請務必確認你的永豐金帳號有 「零股手續費最低 1 元」 的優惠。
如果沒有這個優惠（最低 20 元），頻繁試單會吃掉你的所有獲利。
