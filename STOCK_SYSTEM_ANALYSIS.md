## 📊 系統概覽

### 核心組件:

### 配置檔案:
- `config/stocks.yaml` - 主要配置
- `.env` - API金鑰和環境變數

## 🔄 完整交易流程

### 階段1: 系統初始化
```
1. 啟動 StockRunner (獨立進程)
2. 登入 Shioaji API
3. 載入配置 (stocks.yaml)
4. 初始化 StockMonitor
5. 恢復隔夜持倉 (從交易日誌)
```

### 階段2: 盤前準備 (09:00前)
```
1. 執行日線級別型態掃描 (CANSLIM)
2. 更新大盤多空判斷 (TAIEX EMA60)
3. 下載最新股票資料 (5分鐘K線)
```

### 階段3: 盤中監控 (09:00-13:30)
```
1. 動態篩選觀察清單 (成交量+開盤強勢)
2. 每5分鐘檢查各標的:
   - 取得即時快照 (snapshot)
   - 計算技術指標 (Squeeze, Momentum, KD等)
   - 執行策略邏輯 (mean_reversion等)
   - 產生交易信號
3. 執行交易:
   - 檢查風險限制 (空頭防禦、每日虧損上限)
   - 下單 (整股/零股)
   - 更新持倉狀態
4. 監控掛單:
   - 清理未成交訂單
   - 調整停損停利
```

### 階段4: 盤後處理 (13:30後)
```
1. 平倉所有持倉 (可選)
2. 寫入交易日誌
3. 生成績效報告
4. 準備隔夜資料
```

## 📈 策略邏輯詳解

### 當前配置策略: **均值回歸 (mean_reversion)**
```yaml
策略: mean_reversion
資金: 總額100,000 TWD，每筆20,000 TWD
停損: 5%
停利: 15%
追蹤停損: 2%
空頭防禦: 啟用
```

### 可用策略列表:
1. **mean_reversion** - 布林通道均值回歸
2. **arbitrage** - 整零價差套利
3. **momentum** - 動能突破 (帶量確認)
4. **scout** - 零股偵察兵 (Squeeze Fired + 成交量)
5. **kd_mean_reversion** - KD超賣回歸 + ADX過濾

### 空頭防禦機制 (Bear Defense):
```yaml
啟用: true
大盤EMA長度: 60
最大每日虧損: 3,000 TWD
最大連續虧損: 3次
空頭最大持倉: 1檔
正常最大持倉: 3檔
```

## 🔧 技術指標計算

### 即時計算 (每5分鐘):
1. **Squeeze Indicator** - 擠壓指標
2. **Momentum State** - 動能狀態 (0-3)
3. **Bollinger Bands** - 布林通道
4. **KD Indicator** - 隨機指標
5. **MACD** - 指數平滑異同移動平均線
6. **ADX** - 平均趨向指數
7. **Volume Analysis** - 成交量分析

### 日線級別 (盤前):
1. **CANSLIM Pattern** - 成長股型態
2. **Pivot Points** - 樞軸點計算
3. **Market Regime** - 大盤多空判斷

## ⚙️ 風險管理架構

### 多層次風險控制:
1. **部位限制**:
   - 正常市場: 最大3檔
   - 空頭市場: 最大1檔
   - 每筆交易: 20,000 TWD

2. **虧損控制**:
   - 單筆停損: 5%
   - 每日最大虧損: 3,000 TWD
   - 連續虧損: 最多3次後暫停

3. **市場過濾**:
   - 大盤EMA60判斷多空
   - 成交量確認
   - 流動性檢查

4. **執行安全**:
   - Paper模式優先
   - 訂單超時清理
   - 異常狀況自動暫停
