| broker open-order/fill snapshot → strategy pending state | Broker truth controls whether strategy metadata is restored as pending, terminal, or blocked |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | T | restart rehydration in futures/options monitors | mitigate | Reconcile restored orders against broker open orders/fills before rebuilding pending metadata or positions |
| T-02-05 | D | startup trade path | mitigate | Seed `_pending_lifecycle_orders`, `pending_entry`, and `pending_exit_trade` from restored lifecycle orders so the first tick does not submit a duplicate order |
| T-02-06 | R | recovered fill dedupe | mitigate | Seed `_applied_lifecycle_deals` / equivalent deal tracking from restored fills and preserve original `intent_id`/`order_id`/`deal_id` linkage during startup |
</threat_model>
