<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker refresh payload → `OrderManager` | Untrusted broker status/deal snapshots can overwrite local lifecycle truth |
| normalized audit trail → restart/review tooling | Persisted provenance becomes the basis for operator decisions after incidents |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `core/order_management/order_manager.py` reconciliation | mitigate | Reconcile only onto an existing lifecycle order resolved by local ID or broker IDs and dedupe fills by canonical deal identifiers, not `ordno` alone |
| T-02-02 | R | normalized audit history | mitigate | Append stable `timestamp`/`source`/`reason` fields for every submission, refresh, deal, cancel, reject, expire, and recovery transition |
| T-02-03 | I | broker snapshot adapters | mitigate | Add explicit read-only snapshot methods (`list_open_orders`, `list_trades`, `list_positions`) instead of overloading submit/cancel paths, keeping live order side effects isolated |
</threat_model>
