## 🎯 核心哲學：大賺小賠

```
【傳統思維】
勝率 50% + 盈虧比 1:1 = 打平

【優化思維】
勝率 40% + 盈虧比 2:1 = 賺錢！

關鍵：不在於「對的次數」，而在於「對的時候賺多少」
```

### 數學驗證

**每 10 次交易 (勝率 40%)**:

```
【優化前 - 盈虧比 1.33:1】
獲利：4 次 × 2.0 ATR = +8 ATR
虧損：6 次 × 1.5 ATR = -9 ATR
淨利：-1 ATR ❌

【優化後 - 盈虧比 2:1】
獲利：4 次 × 2.0 ATR = +8 ATR
虧損：6 次 × 1.0 ATR = -6 ATR
淨利：+2 ATR ✅
```

**結論**: 同樣 40% 勝率，從虧損轉為獲利！

## ⚙️ 參數調整詳解

### 1. 止損/止盈 (核心調整)

```yaml
# 優化前
stop_atr_mult: 1.5
take_profit_atr_mult: 2.0
盈虧比：1.33:1

# 優化後
stop_atr_mult: 1.0      # 緊止損
take_profit_atr_mult: 2.0  # 合理止盈
盈虧比：2.0:1 ✅
```

**舉例**:
```
進場：22000, ATR=50

【優化前】
止損：22075 (虧 75 點)
止盈：21900 (賺 100 點)
盈虧比：1.33

【優化後】
止損：22050 (虧 50 點) ← 小賠
止盈：21900 (賺 100 點) ← 大賺
盈虧比：2.0 ✅
```

### 2. 進場門檻 (配套加嚴)

為了避免緊止損被洗出場，**必須提高進場品質**:

```yaml
# VWAP 距離 (更接近 VWAP)
max_vwap_dist_atr: 0.8 → 0.5  # ⬇️ 37% 更嚴格

# 動能門檻 (更強動能)
min_mom_velo_bearish: -5.0 → -8.0  # ⬆️ 60% 更嚴格

# ADX 上限 (更純的 WEAK)
max_adx: 22.0 → 20.0  # ⬇️ 更嚴格
```

**效果**:
- ✅ 進場點更優質 (減少被洗)
- ✅ 動能更強 (快速脫離成本區)
- ⚠️ 進場次數可能減少 (預期 -30%)

### 3. 時間止損 (加速出場)

```yaml
time_stop_minutes: 20 → 15  # ⬇️ 縮短 25%
```

**邏輯**:
- 如果 15 分鐘內無法獲利，說明進場時機不對
- 早點出場，避免小虧變大虧
- 釋放資金，等待更好的機會

## ⚠️ 風險與挑戰

### 風險 1: 進場次數減少

**預期**: 從每晚 3-5 次 → 2-3 次 (-30%)

**對策**:
- ✅ 接受「少即是多」哲學
- ✅ 品質 > 數量
- ✅ 寧可錯過，不要做錯

### 風險 2: 緊止損被洗

**預期**: 可能遇到「剛止損就反轉」

**對策**:
- ✅ 嚴格進場條件 (減少被洗機率)
- ✅ 接受這是交易成本
- ✅ 長期來看，盈虧比優勢會覆蓋

### 風險 3: 止盈太難達到

**預期**: 2.0 ATR 可能太遠

**對策**:
- ✅ 觀察實際數據 (如果 <30% 達成率，調整為 1.8 ATR)
- ✅ 考慮分段止盈 (1.0 ATR 減碼 50%, 2.0 ATR 全平)
