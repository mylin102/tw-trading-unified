# 期權與期貨交易系統一致性問題 GSD 分析報告

## 執行摘要
本報告使用 GSD (Gather-Sort-Decide) 方法分析並解決了台灣期權與期貨交易系統中的數據不一致問題。問題包括期權交易記錄顯示不一致、期貨交易顯示缺失、夜盤日期邏輯錯誤等。

## 1. GATHER 階段：問題收集

### 1.1 期權交易問題
- **Dashboard顯示**: 4筆期權交易 (17:52進場, 17:53出場)
- **CSV記錄**: 12筆記錄 (6 THETA_ENTRY + 6 THETA_EXIT)
- **JSON訂單**: 2筆訂單，時間戳不一致 (04:51 UTC vs 17:52本地時間)
- **總覽頁面**: 顯示「進場12筆」
- **PnL計算**: Theta策略顯示負PnL，但應為信用策略
- **價格記錄**: 所有THETA_ENTRY記錄價格為0

### 1.2 期貨交易問題
- **訂單狀態**: 3筆期貨訂單 (全部成交)
- **交易記錄**: Dashboard缺少交易記錄和未實現損益欄位
- **總覽顯示**: 「今日交易:0」但實際有交易
- **文件命名**: `TMF_20260415_trades.csv` 存在，但Dashboard尋找 `TMF_20260416_trades.csv`

### 1.3 夜盤日期邏輯問題
- **當前時間**: 2026-04-16 05:14 (週四凌晨)
- **get_trade_day**: 返回 `20260415` (正確，屬於前一日夜盤)
- **get_session_date_str**: 返回 `20260416` (用於Dashboard顯示)
- **文件不匹配**: 交易記錄文件使用 `20260415`，Dashboard尋找 `20260416`

## 2. SORT 階段：問題分類與根本原因分析

### 2.1 期權問題根本原因
1. **價格記錄錯誤**: `live_options_squeeze_monitor.py` 第2338行，`entry_price = float(pos.net_credit) if pos.net_credit else 0.0` 未檢查 `net_credit > 0`
2. **退出識別缺失**: `exit_keywords` 列表缺少 "THETA_EXIT"
3. **淨信用驗證**: `theta_gang.py` 未驗證 `net_credit <= 0` 的情況
4. **歷史數據損壞**: 24筆THETA_ENTRY記錄價格為0

### 2.2 期貨問題根本原因
1. **日期邏輯不一致**: `get_trade_day` 和 `get_session_date_str` 使用不同的夜盤日期歸屬邏輯
2. **文件命名不匹配**: Dashboard使用 `get_session_date_str` 尋找文件，但交易記錄使用 `get_trade_day` 命名文件
3. **顯示邏輯錯誤**: Dashboard顯示「今日交易」基於日曆日，而非交易日

### 2.3 性能與優化問題
1. **頻繁文件I/O**: 夜盤tick數據導致頻繁CSV寫入
2. **缺少緩衝機制**: 每次交易都直接寫入文件
3. **缺少結算價記錄**: 跨日持倉未實現損益應使用交易所結算價計算

## 3. DECIDE 階段：解決方案與實施

### 3.1 期權問題解決方案 (已實施)
1. **修復價格記錄**: 在 `live_options_squeeze_monitor.py` 添加 `net_credit > 0` 檢查
2. **添加退出關鍵字**: 將 "THETA_EXIT" 加入 `exit_keywords` 列表
3. **增強淨信用驗證**: 在 `theta_gang.py` 添加 `net_credit <= 0` 檢查和警告
4. **修復歷史數據**: 創建 `repair_csv_prices.py` 修復24筆THETA_ENTRY記錄

### 3.2 期貨問題解決方案 (已實施)
1. **統一日期邏輯**: 修改 `get_trade_day` 函數，整合參考資料中的 `get_trading_date` 邏輯
   - 15:00之後 (夜盤開始): 週五晚跳週一，其他日跳隔日
   - 00:00-08:00 (夜盤後半段): 週六凌晨跳週一，其他日保持當日
   - 08:00-14:59 (日盤): 保持當日
2. **Dashboard優化**: 添加 `TRADE_DATE_STR` 變數，使用 `get_trade_day` 確保文件一致性
3. **文件查找邏輯**: 修改 `load_futures_trades()` 先嘗試 `TRADE_DATE_STR`，再嘗試 `DATE_STR`

### 3.3 性能優化方案 (已實施)
1. **緩衝寫入機制**: 在 `DataStorage` 類別添加 `trade_buffer`、`buffer_size=1000`、`flush_interval=60`
2. **批量寫入**: 實現 `_check_and_flush_buffer()` 和 `_flush_buffer()` 方法
3. **追加模式CSV**: 使用 `_save_trades_csv_append()` 避免重複寫入標題

### 3.4 顯示邏輯優化 (建議)
1. **修改顯示文字**: 將「今日交易」改為「本交易日交易」
2. **添加交易日標示**: 在Dashboard顯示當前交易日日期
3. **區分日曆日與交易日**: 明確標示交易日的時間範圍
