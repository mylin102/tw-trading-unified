# CB / CBAS Regime Signal — Planning Document

## Goal

Add convertible bond market breadth as a second-layer sentiment signal, sitting alongside ETF Regime.

```
ETF Regime = capital allocation direction (macro)
CB / CBAS  = risk appetite, speculative flow, leverage sentiment (micro)
```

## Architecture

```
tw-canslim-web
  → TPEX CB daily quotes
  → data/cb_regime.json

tw-trading-unified
  → core/cb_regime_consumer.py
  → threshold / size modifier only
  → no direct trade signals
```

## Data Sources (priority order)

| Source | Type | Notes |
|--------|------|-------|
| TPEX OpenAPI | Primary | Official CB daily trading data |
| TPEX CB daily quotes page | Fallback | HTML table at tpex.org.tw |
| FinMind TaiwanStockConvertibleBondDaily | Last resort | Backer/sponsor only, not free |

## v1 Features

Simplified — no conversion premium calculation yet:

| Feature | Formula |
|---------|---------|
| `cb_breadth` | Rising CB count / total traded CB count |
| `cb_volume_spike` | Today turnover / 20-day avg turnover |
| `cb_hot_count` | Count of CBs with price change > 3% + volume up |
| `cb_momentum_5d` | Equal-weighted CB 5-day return |
| `cb_equity_lead` | CB return - underlying stock return (avg) |

## Regime Classification

| Regime | Signal |
|--------|--------|
| `CB_RISK_ON` | breadth > 0.6, volume spike > 1.2, hot_count high |
| `CB_SPECULATIVE` | volume spike high, equity_lead positive, but moderate breadth |
| `CB_DEFENSIVE` | breadth < 0.4, volume low, equity_lead negative |
| `CB_STRESS` | breadth very low, volume spike (panic), hot_count negative |

## Phases

| Phase | What | Depends on |
|-------|------|------------|
| 1 | CB daily quotes → breadth / volume_spike / hot_count | TPEX data access confirmed |
| 2 | Add equity_lead (CB vs stock) | Underlying stock mapping |
| 3 | Conversion premium / fair value | Conversion price data |
| 4 | Futures/options modifier integration | Phase 1-3 stable |

## Key Principles

- CB regime is a **risk appetite thermometer**, not a trade signal
- Use existing ETF Regime consumer pattern (same modifier structure)
- Degrade gracefully on data failure
- v1 = no conversion math, just price + volume
