## Testing

### Fast feedback loop (local development)

```bash
# Run all tests quickly (no coverage, stop on first failure)
./scripts/pytest-quick.sh

# Filter by keyword
./scripts/pytest-quick.sh -k "squeeze"
./scripts/pytest-quick.sh -k "orb or mts"

# Run a specific directory
./scripts/pytest-quick.sh tests/strategies/
./scripts/pytest-quick.sh tests/contracts/

# Pass any pytest args
./scripts/pytest-quick.sh -k "spread" --tb=long
```

The `pytest-quick.sh` script automatically adds:
- `-x` (stop on first failure)
- `--tb=short` (concise tracebacks)
- `-n auto` (parallel workers if pytest-xdist installed)
- No coverage instrumentation

Set `MAX_FAILURES=10` or `WORKERS=2` via environment to customise.

### Full coverage test (before push / CI)

```bash
# Run everything with coverage report
pytest --cov=. --cov-report=term-missing

# Run a subset with coverage
pytest --cov=. tests/contracts/

# Quick alias: coverage with html report
pytest --cov=. --cov-report=html --cov-report=term-missing
```

Coverage reports are also generated by CI in `.github/workflows/ci-basic.yml` via explicit `--cov=` flags. Removing coverage from default `pytest` does not affect CI gating.
