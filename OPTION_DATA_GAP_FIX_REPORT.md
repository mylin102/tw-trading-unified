# 期權kbar缺資料問題修復報告

## 問題概述
期權交易系統在kbar資料缺失時會產生驟降的kbar，導致交易策略誤判，這是一個致命的系統性風險。

## 修復內容

### 1. 資料完整性驗證系統
在 `live_options_squeeze_monitor.py` 中添加了完整的資料驗證機制：

#### 新增方法：
- `_validate_kbar_data()`: 驗證kbar資料完整性
- `_fill_small_kbar_gaps()`: 填充小間隔的資料缺口  
- `_validate_and_fill_kbar_gaps()`: 完整的驗證和填充流程

#### 驗證項目：
- **資料完整性**: 檢查是否為空、最少kbar數量（≥30根）
- **時間連續性**: 檢查時間間隔是否正常（5分鐘±1分鐘）
- **缺口檢測**: 檢測並標記資料缺口
- **價格有效性**: 檢查價格是否為正數、無NaN值
- **欄位完整性**: 確保所有必要欄位存在

### 2. 智慧缺口處理策略

#### 小缺口處理（≤15分鐘）：
- 自動檢測並填充缺失的kbar
- 使用前向填充（最多填充3根kbar）
- 保持資料連續性，避免技術指標跳躍

#### 中缺口處理（15-30分鐘）：
- 部分填充可恢復的缺口
- 記錄警告日誌
- 可選：降低交易頻率或收緊風險控制

#### 大缺口保護（＞30分鐘）：
- 暫停策略執行
- 發出高風險警報
- 等待資料恢復後再繼續

### 3. 策略執行保護層
修改 `fetch_live_signal()` 方法：
- 在獲取資料後立即進行驗證
- 資料無效時嘗試備用資料源
- 資料無法修復時跳過信號生成
- 詳細日誌記錄所有處理步驟

## 修復效果

### 測試結果：
1. **正常資料**: ✓ 通過驗證，正常使用
2. **小缺口資料（≤15分鐘）**: ✓ 自動填充，繼續使用
3. **中缺口資料（15-30分鐘）**: ⚠️ 部分填充，可能暫停策略  
4. **大缺口資料（＞30分鐘）**: ✗ 暫停策略，等待恢復
5. **異常資料（NaN值等）**: ✗ 拒絕使用，保護策略

- **系統性風險**: 顯著降低

```python
# 資料驗證核心邏輯
def _validate_kbar_data(self, df):
    if df is None or df.empty:
        return False, "資料為空"
    
    # 檢查最少kbar數量
    if len(df) < 30:
        return False, f"資料不足: {len(df)}根"
    
    # 檢查時間間隔
    time_diffs = df.index.to_series().diff().dt.total_seconds() / 60
    if len(time_diffs) > 1:
        max_gap = time_diffs.iloc[1:].max()
        if max_gap > 30:  # 最大允許30分鐘缺口
            return False, f"資料缺口過大: {max_gap:.0f}分鐘"
    
    return True, "資料完整"
```

### 2. 監控日誌：
