# 策略優化提案：Squeeze 失敗反向過濾策略 (Squeeze Failure Counter-Strategy)

## 1. 背景與漏洞診斷
經過對 `TMF` (微台指) 5 分鐘週期的深度回測與審計，發現原始 Squeeze 突破策略存在以下漏洞：
*   **假突破頻繁**：台指期在非趨勢盤（特別是夜盤）具有強烈的「均值回歸」特性。Squeeze 釋放後的爆發力往往不足以支撐長波段。
*   **盈虧比失衡**：為了提高勝率，目前最優參數為 `ATR 3.0x` 停損搭配 `40點` 獲利，這導致盈虧比僅約 **1:0.4**，在統計學上屬於極難獲利的結構。
*   **反應滯後**：當多週期共振達成時，價格往往已處於波動率末端。

---

## 2. 核心邏輯：反向操作 (Mean Reversion after Failed Breakout)
與其強行捕捉難以持續的突破，不如利用「突破失敗」後的回抽力量進行反向操作。

### A. 偵測階段 (Detection)
1.  **Squeeze 釋放**：`sqz_on` 由 `True` 轉為 `False` (Fired)。
2.  **方向確認**：`momentum` > 0 為多頭突破預期，`momentum` < 0 為空頭突破預期。

### B. 失敗驗證 (Failure Validation)
在 Squeeze 釋放後的 **3-5 根 K 棒** 內，若滿足以下任一條件，判定為「突破失敗」：
1.  **價格未創新高/低**：多頭突破但價格未能收在 `recent_high` 之上；空頭突破但未能收在 `recent_low` 之下。
2.  **動能斜率 (Velo) 反轉**：`mom_velo` 快速歸零或反向噴發。
3.  **VWAP 拒絕**：多頭突破後迅速跌破 `VWAP` 線。

### C. 進場機制 (Entry)
當判定突破失敗後，立即執行 **反向進場**：
*   **原本看多但失敗** -> 改為 **SELL SHORT**。
*   **原本看空但失敗** -> 改為 **BUY LONG**。

---

## 3. 獲利與風險控制
反向策略的核心優勢在於「目標明確」且「風險可控」。

*   **獲利目標 (Take Profit)**：
    *   **首要目標**：回歸 `VWAP` 線。
    *   **最終目標**：觸碰 Bollinger Band 的另一側（對邊）。
*   **停損機制 (Stop Loss)**：
    *   設置於導致突破失敗的那根 K 棒的高/低點（極窄停損）。
    *   或者使用 `1.0x ATR`，因為均值回歸若未立即發生，則代表趨勢可能真啟動了。

---

## 4. 預期改進指標 (Metrics Expectation)
| 指標 | 原始突破策略 (優化後) | 反向過濾策略 (預期) |
| :--- | :--- | :--- |
| **勝率 (Win Rate)** | 57% | 65% - 70% |
| **獲利因子 (PF)** | 0.86 | **1.25 - 1.50** |
| **盈虧比 (R:R)** | 1 : 0.4 | **1 : 1.2** |
| **交易頻率** | 高 (易磨損) | 中 (精確捕獲失敗) |

---

## 5. 實作建議
建議在 `strategies/futures/monitor.py` 中新增一個 `counter_mode` 開關。
*   當 `counter_mode = True` 時，原有的 `_execute_trade` 進場訊號將被延後 3 棒執行。
*   若 3 棒內出現上述「失敗驗證」訊號，則執行反向單。
