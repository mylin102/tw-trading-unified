# 期權kbar缺資料導致驟降問題分析報告

## 問題描述
期權交易系統在kbar資料缺失時會產生驟降的kbar，這可能導致交易策略誤判。

## 技術分析

### 1. 資料來源問題
期權監控器依賴兩個主要資料來源：
- `_fetch_today_futures_bars()`: 從Shioaji API獲取當日期貨kbar
- `_get_mtx_tick_bars_df()`: 從tick資料構建kbar

### 2. 缺資料的具體場景
1. **API調用失敗**: Shioaji API返回空資料或錯誤
2. **網路中斷**: 連線問題導致資料獲取失敗
3. **市場休市**: 凌晨時段(02:00-05:00)無資料
4. **合約切換**: 到期合約切換時資料不連續

### 3. 對策略的具體影響

#### 3.1 技術指標計算錯誤
- **EMA/MA指標**: 缺資料導致移動平均計算跳躍
- **ATR波動率**: 波動率計算異常，影響止損距離
- **布林帶**: 帶寬計算錯誤，誤判突破信號
- **擠壓指標**: 布林帶與KC通道關係誤判

#### 3.2 多時間框架對齊錯誤
V2策略需要5m、15m、1h時間框架對齊：
- 任一時間框架缺資料都會影響對齊分數
- 可能產生虛假的MTF對齊信號
- 錯誤的趨勢方向判斷

#### 3.3 風險管理失效
- **止損計算**: ATR因缺資料計算錯誤
- **倉位管理**: 風險暴露計算不準確
- **資金管理**: 保證金需求計算錯誤

## 實際風險評估

### 高風險場景
1. **虛假突破信號**
   - 缺資料導致價格看似突破關鍵水平
   - 策略誤判為趨勢開始而進場
   - 潛在損失: 1-2% per trade

2. **擠壓狀態誤判**
   - 波動率計算錯誤導致擠壓狀態誤判
   - 錯過真正突破時機或提前進場
   - 潛在損失: 機會成本 + 錯誤進場損失

3. **凌晨時段交易**
   - 02:00-05:00資料不穩定
   - 夜盤策略可能產生錯誤信號
   - 潛在損失: 系統性風險

### 中風險場景
1. **止損設置錯誤**
   - ATR計算異常導致止損過近或過遠
   - 潛在損失: 2-5% per trade

2. **多策略協同錯誤**
   - 多個策略同時因缺資料誤判
   - 超額持倉風險
   - 潛在損失: 組合風險

## 解決方案

### 立即措施（高優先級）

#### 1. 資料完整性檢查
```python
def validate_kbar_data(df, min_bars=30, max_gap_minutes=15):
    """驗證kbar資料完整性"""
    if df is None or len(df) < min_bars:
        return False, "資料不足"
    
    # 檢查時間間隔
    time_diffs = df.index.to_series().diff().dt.total_seconds() / 60
    max_gap = time_diffs.max()
    
    if max_gap > max_gap_minutes:
        return False, f"資料缺口過大: {max_gap:.0f}分鐘"
    
    return True, "資料完整"
```

#### 2. 資料填充機制
```python
def fill_small_gaps(df, max_gap_minutes=15):
    """填充小間隔的資料缺口"""
    if df is None or df.empty:
        return df
    
    # 重新採樣並前向填充小缺口
    resampled = df.resample('5min').asfreq()
    small_gaps = resampled.isna().sum().sum()
    
    if small_gaps > 0:
        df_filled = resampled.ffill(limit=3)  # 最多填充3根kbar
        return df_filled
    
    return df
```

## 結論

期權kbar缺資料確實會導致交易策略誤判，主要風險包括：
1. 技術指標計算錯誤產生虛假信號
