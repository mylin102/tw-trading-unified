## 1. 遷移概述

### 變更內容
將股票系統的推薦名單來源從 `squeeze-tw-screener` 改為 `tw-canslim-web`。

### 變更原因
- **策略升級**: 從技術面選股 (Squeeze) 升級為基本面+技術面整合選股 (CANSLIM)
- **數據品質**: CANSLIM 提供更全面的基本面分析
- **整合性**: 與現有的 CANSLIM 信號系統更好地整合

## 2. 技術變更詳情

### 修改的檔案
1. **`scripts/sync/sync_watchlist.py`** - 主要同步腳本
   - 數據來源: `../squeeze-tw-screener/recommendations.csv` → `../tw-canslim-web/docs/data.json`
   - 選股邏輯: 從追蹤狀態過濾 → 基於 CANSLIM 分數和條件數量
   - 輸出格式: 更詳細的選股資訊顯示

### 新的選股邏輯
```python
# 選股條件
1. CANSLIM 分數 >= 70
2. 至少符合 5/7 個 CANSLIM 條件
3. 按分數排序，取前 10 檔
```

### 數據結構對比
| 項目 | 舊系統 (Squeeze) | 新系統 (CANSLIM) |
|------|------------------|------------------|
| 數據格式 | CSV | JSON |
| 選股指標 | 技術面狀態 | 基本面+技術面綜合評分 |
| 條件數量 | 1個 (追蹤狀態) | 7個 (C/A/N/S/L/I/M) |
| 評分系統 | 無 | 0-100 分 |
| 更新頻率 | 每日 | 每日 |

## 4. 系統影響分析

### 正面影響
1. **選股品質提升**: 從純技術面升級為基本面+技術面綜合分析
2. **風險控制改善**: CANSLIM 條件提供多維度風險過濾
3. **系統整合**: 與現有 CANSLIM 信號系統無縫整合
4. **數據豐富度**: 獲得更多基本面和分析數據

### 潛在風險
1. **數據依賴**: 依賴 `tw-canslim-web` 的數據更新頻率
2. **條件嚴格**: CANSLIM 條件可能過於嚴格，減少選股數量
3. **過渡期**: 舊有 squeeze 股票可能逐漸被替換

### 緩解措施
1. **備份機制**: 保留舊有 squeeze 股票在監控名單中
2. **監控機制**: 每日檢查 CANSLIM 數據更新狀態
3. **回退方案**: 如有問題可暫時切換回舊系統
